Perkowitz, M., and O. Etzioni. "Adaptive Sites: Automatically Learning from User Access Patterns." 1997. Sixth International World Wide Web Conference. 27 August 2002 <http://www.scope.gmd.de/info/www6/posters/722/index.html>.

(also available at http://www.cs.washington.edu/research/adaptive/WWW6/)

Summary written by Anitra

Adaptive Websites:

  • deals with the idea of an "assistant" to make suggestions or small changes to a particular site, including:
    • "promotion" - make a link easier to find by putting it (or a copy of it) closer to the front page of the site or closer to the top of the page
    • "hilighting" - changing the color/graphic/font of a link to emphasize it. A "lightweight alteration".
    • "linking" - placing a link to connect two pages that were previously unconnected - deals with conceptual & semantic relations existing in users' minds.
    • "clustering" - taking a group of related pages and creates a new page to link to all of them. Recognises similarities in documents based on filename, location/path, and user access. May even be possible to find relations based on content (index whole files).
  • also addresses the question "Why are adaptive websites needed?"
    • can customize for different types/groups of users, not just individual users or all users as a whole.
    • helps to recognise when user perceptions of the site differ from how the site was designed.
    • user needs change with time - so can the website.

Important quote: "Although a service like WebThreads can be extremely useful, path data can also be heuristically inferred from standard server logs. If we assume that accesses originating from the same machine at around the same time correspond to the visit of a single user making a coherent series of page visits, we can record sequences of pages accessed by individual visitors. We can construct a graph of the site, where nodes represent pages and directed arcs represent links, and match these sequences against the graph to determine the full path followed. "
