Source:

Langley, P. and Hirsh, H. User Modeling and Adaptive Interfaces. Seventeenth National Conference on AI.

Summary by Mike

  • personalization requires some model of the user's behavior
  • personalized systems depend on some stored model of the user to predict his preferences and behavior
  • the problem of learning individual models: (adaptive user interfaces)
    • given: a set of tasks that require some user decision
    • given: descriptions for each of these tasks
    • given: traces of the users decisions on each task
    • find: mapping from task features to user decisions
  • questionnaire(tediuos?) vs. traces(less obtrusive)
  • definition "a software artifact that reduces user effort by aquiring a user model based on past interaction"
  • steps in developing adaptive interfaces:
    • problem formulation
      • given: some task that an intelligent assistant could aid
      • find: some formulation that lets the assistant improve its performance by learning a user model from experience
      • design choices: the aspect of user behavior tp be predicted, the level of desciption (what constitutes a training case)
    • representation engineering
      • given: a formulation of some task in machine learning terms
      • find: some representation for behavior and user models that makes the learning task tractable
      • design choices: information to be used when predicting behavior, internal encoding of that information in the system (i.e. attribute-value)
    • collecting user traces
      • given: a problem formulation in terms of machine learning and a representation of user behavior
      • find: an effective way to collect traces of this behavior
      • design chioces: how to transform these traces into training instances, what action the user must make to generate the traces (i.e. ideally no extra user effort)
    • using the learned model
      • given: an approach to learning a user model for some task
      • find: some way to invoke the model that helps the user perform the task more effectively
      • design choices: when and how to present the model's predictions to the user, how to handle cases in which these predictions are wrong
    • gaining user acceptance
      • given: a complete user interface for some task, ways to get people to try the system and to become long term users
  • examples of adaptive interfaces are presented for the rest of the tutorial:
    • NewsWeeder, NewsDude, DailyLearner, MailCat, EmailValet...
